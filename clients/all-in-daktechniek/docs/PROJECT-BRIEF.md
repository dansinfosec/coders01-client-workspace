## Opdracht

De bestaande website volledig **moderniseren en ombouwen** tot een geloofwaardige,
mobiel-sterke leadgeneratie-site — géén generieke AI-website, maar passend bij het
bestaande bedrijf.

## Doelen (uit opdracht)

1. Moderne, betrouwbare uitstraling voor een professioneel dakdekkersbedrijf.
2. Sterke mobiele ervaring.
3. Duidelijke uitleg van alle dakwerkzaamheden.
4. Bezoekers snel laten **bellen / WhatsAppen / offerte aanvragen**.
5. Projectfoto's / uitgevoerd werk overtuigend presenteren.
6. Lokale SEO verbeteren.
7. Meer offerteaanvragen genereren.

## Harde randvoorwaarden (Coders01-standaard + opdracht)

- **Geen bedrijfsfeiten verzinnen** — geen certificeringen, garanties, vestigings-
  plaatsen, jaren ervaring, klantbeoordelingen of claims die niet aantoonbaar op de
  huidige site staan. Onbekende waarden blijven leeg met `TODO: Confirm with client`.
- **Niet zelfstandig extern onderzoek doen** naar het bedrijf. Alleen wat op
  all-indaktechniek.nl staat is bron. Ontbrekende info → markeren als "nodig van Borre".
- Geen publicatie of deployment zonder expliciete opdracht.
- Geen andere klantprojecten wijzigen; gedeelde `automation/` of `templates/` alleen
  aanpassen indien echt noodzakelijk.
- Niet committen/pushen zonder expliciete opdracht.

## Technische standaard (overgenomen van de workspace)

React 18 · Vite 5 · **strict TypeScript** · Tailwind CSS 3 (semantische tokens via
CSS-variabelen) · React Router 6 · React Hook Form + Zod · lucide-react ·
react-helmet-async. Statische SPA, `dist/`, Vercel-rewrite. Referentieproject:
`clients/roofing-center-react-mvp/`. Zie `Coders01 - Technical Standards` in Obsidian.
